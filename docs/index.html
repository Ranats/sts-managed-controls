Capabilities

Built for controlled single-player testing loops.

Live State Probe

Read floor, ascension, HP, block, gold, and energy through the managed snapshot path without relying on OCR.

Runtime Actions

Set gold directly, maintain block or energy, and apply powers through a bridge-based in-process workflow.

Catalog Search

Search cards, powers, and relics discovered from sts2.dll before triggering runtime actions.

Desktop UI

Use a dedicated local window instead of repeating CLI calls or relying on an in-game overlay.

Fast Test Loops

Set combat-start auto powers and create controlled scenarios for mod validation, balance iteration, and debugging.

Experimental Paths

Deck replacement, relic obtain, map jump, and dynamic-var tuning are present but still marked experimental.

Limits

What to expect

  • Local single-player use only
  • Back up saves if you care about them
  • Experimental bridge actions can leave unusual save or combat state
  • Game updates can break reflection-based bridge calls
